# Voxel Buddy

> Animated desktop communication companion with Slack Socket Mode and Gmail integrations

![Status](https://img.shields.io/badge/status-active%20development-blue)
![Electron](https://img.shields.io/badge/Electron-33-47848F?logo=electron)
![Platform](https://img.shields.io/badge/platform-macOS%20%7C%20Windows%20%7C%20Linux-lightgrey)
![License](https://img.shields.io/badge/license-MIT-green)

Voxel Buddy is a frameless, always-on-top Electron desktop app that plants an animated mascot character at the bottom of your screen. When a Slack DM or @mention arrives — or a new email lands in your Gmail inbox — your companion wakes up, wiggles, and floats a speech bubble with a preview and a one-click reply field. No alt-tab required.

Built on Electron 33, React, and Vite, it ships as a native distributable for macOS, Windows, and Linux.

---

## Features

- **Animated mascot overlay** — a full-screen transparent Electron window sits over your desktop. The mascot renders inside it and is normally click-through so it never steals focus from your other apps.
- **Airie character** — teal robot-with-wings mascot with a 12-state sprite sheet: `idle` (hovering fly-strip), `walk` (2-frame walk-strip), `alert`, `wave`, `sleep`, `happy`, `meditate`, `love`, `confused`, `determined`, `surprised`, and `thinking`. New characters can be added by dropping assets into `src/assets/` and registering them in `MascotRenderer`.
- **Buddy character** — classic blob companion available as an alternative in settings.
- **Table-driven FSM behavior loop** — a `requestAnimationFrame` game loop drives a finite state machine with per-state `onEnter` / `onTick` / `onExit` hooks. Airie wanders the screen, glances left/right, occasionally meditates or naps, and reacts to notifications in real time.
- **Idle sleep / battery-friendly polling** — after 20 seconds of inactivity the FSM enters `sleep` and drops from rAF to a 500 ms `setTimeout` poll, saving CPU when you step away.
- **Slack Socket Mode integration** — long-lived WebSocket connection via `@slack/socket-mode`. Receives DMs (`message.im`) and `app_mention` events instantly with no HTTP polling. Reconnects automatically with exponential backoff capped at 30 seconds.
- **In-app Slack replies** — each notification carries a reply handler keyed by UUID. Type a reply in the speech bubble and hit Enter; the message posts via `chat.postMessage` in the correct thread. Airie enters a `thinking` pose while the IPC round-trip runs and plays a success trill on completion.
- **Gmail polling integration** — polls `is:unread in:inbox` every 60 seconds via the Gmail REST API. Deduplicates by message ID so the same email never fires twice. Dismissing a Gmail notification marks the message as read (removes the `UNREAD` label).
- **Google OAuth flow** — a local HTTP server on `localhost:42813` opens the browser to Google's consent screen, captures the authorization code with CSRF state validation, and exchanges it for tokens — entirely in-process, no external redirect service needed.
- **OS-keychain token encryption** — all credentials (Slack app token, bot token, Gmail access token, refresh token, client ID, client secret) are encrypted at rest using `electron.safeStorage` before being written to `~/.voxelbuddy/settings.json`. The renderer always receives decrypted tokens; it never sees ciphertext.
- **AI reply suggestions** — an optional Claude API key unlocks 3 one-click reply chips in every speech bubble, generated by `claude-haiku-3-5-20241022`. Suggestions fail silently if the key is absent or the network request errors.
- **Synthesized notification sounds** — a gentle two-tone chime on arrival and a three-note trill on successful send, generated entirely with the Web Audio API. No audio files required.
- **Right-click context menu** — right-click the mascot to wave, meditate, sleep, fire a test notification, mute/unmute, open settings, or quit.
- **Draggable mascot** — left-click and drag to reposition. The FSM transitions to `happy` on drop. A 4 px movement threshold separates a drag from a click; the click that ends a drag is swallowed so Airie doesn't wave unexpectedly.
- **Double-click Easter egg** — double-clicking within 400 ms triggers the `love` heart animation.
- **Live connection status badges** — the settings panel shows a real-time color-coded indicator (`connecting` / `connected` / `reconnecting` / `disconnected` / `error`) for both Slack and Gmail, pushed from the main process via IPC.
- **Mute toggle** — muted mode lets Airie keep wandering while suppressing speech bubbles. Persisted to disk and reflected in the system tray menu.
- **System tray** — menu for mute, launch-at-login, settings, and quit. Uses a macOS template icon for automatic dark/light adaptation.
- **Launch at login** — one-click toggle via `app.setLoginItemSettings`, launching hidden.
- **Multi-monitor support** — the overlay window auto-resizes when displays are added, removed, or reconfigured.
- **Auto-update** — `electron-updater` checks for updates on launch and installs them on next quit.
- **First-run onboarding card** — appears above the mascot when no Slack tokens are configured, with inline scope requirements and direct links to `api.slack.com/apps`.
- **Error boundary** — a React error boundary catches renderer crashes and renders an inline recovery button so the mascot can restart without relaunching the app.
- **Vitest test suite** — unit tests for the store, integrations, Gmail adapter, and Airie component.

### IPC channels

The `contextBridge` in `preload.js` exposes all communication under a `window.buddy` object. The registered channels are:

| Channel | Direction | Purpose |
|---|---|---|
| `buddy:notification` | main → renderer | Push a new notification object |
| `buddy:send-reply` | renderer → main | Post a Slack reply; returns `{ ok, error? }` |
| `buddy:dismiss-notification` | renderer → main | Remove the reply handler for a notification |
| `buddy:get-settings` | renderer → main | Load decrypted settings from disk |
| `buddy:save-settings` | renderer → main | Encrypt tokens and write settings to disk |
| `buddy:set-muted` | renderer → main | Persist mute state and broadcast back |
| `buddy:mute-changed` | main → renderer | Notify renderer of mute state change |
| `buddy:slack-status` | main → renderer | Live Slack connection status update |
| `buddy:gmail-status` | main → renderer | Live Gmail connection status update |
| `buddy:gmail-auth-start` | renderer → main | Initiate Gmail OAuth flow |
| `buddy:gmail-disconnect` | renderer → main | Revoke Gmail tokens and stop adapter |
| `buddy:gmail-mark-read` | renderer → main | Remove UNREAD label from a Gmail message |
| `buddy:restart-integrations` | renderer → main | Stop and restart all adapters |
| `buddy:open-settings-window` | renderer → main | Open the settings BrowserWindow |
| `buddy:mouse-over-mascot` | renderer → main | Click-through refcount delta `{ delta: +1|-1 }` or `{ reset: true }` |
| `buddy:quit` | renderer → main | Quit the app |
| `buddy:set-login-item` | renderer → main | Enable/disable launch at login |
| `buddy:get-login-item` | renderer → main | Read current login-item state |
| `buddy:open-external` | renderer → main | URL-validated `shell.openExternal` |

### Click-through management

The main overlay window starts with `setIgnoreMouseEvents(true, { forward: true })` so mouse events pass through to apps underneath. When the cursor enters any interactive region (mascot, speech bubble, context menu), the renderer sends a `{ delta: +1 }` IPC message; on leave it sends `{ delta: -1 }`. The main process keeps a refcount and only toggles the window's ignore-state at the 0 ↔ >0 boundary. Window blur and document visibility-hide force a hard reset to 0.

### Slack auth flow

1. User enters App Token (`xapp-1-...`) and Bot Token (`xoxb-...`) in the Settings panel.
2. On save, tokens are encrypted with `electron.safeStorage` and written to `~/.voxelbuddy/settings.json`.
3. On next launch (or after "Reconnect"), `createSlackAdapter` is called with decrypted tokens.
4. `SocketModeClient.start()` opens a WebSocket to Slack's Socket Mode endpoint.
5. `message` events with `channel_type: im` and `app_mention` events are converted to notifications. Bot messages are filtered out.
6. Each notification gets a `replyFn` keyed by UUID. When the user sends a reply the handler calls `chat.postMessage` in the correct thread (`thread_ts` preserved).

### Gmail auth flow

1. User clicks **Connect Gmail** in Settings and enters OAuth 2.0 Client ID and Client Secret from Google Cloud Console.
2. Main process generates an auth URL with `gmail.readonly` + `gmail.modify` scopes and a random CSRF state token, then spins up a local HTTP server on port 42813.
3. `shell.openExternal` opens the URL in the default browser.
4. After the user grants consent, Google redirects to `http://localhost:42813/oauth/google/callback?code=...&state=...`.
5. Main validates the CSRF state, exchanges the code for tokens, fetches the user's email address via `gmail.users.getProfile`, and writes encrypted credentials to disk.
6. The Gmail adapter immediately polls for unread inbox messages, then recurs every 60 seconds.

## Getting Started

### Prerequisites

- **Node.js 20+** (LTS recommended)
- **npm 10+**
- A Slack workspace where you can create and install apps
- A Google Cloud project with the Gmail API enabled _(optional — for Gmail integration)_

### Installation

```bash
git clone https://github.com/rcasto123/voxel-buddy.git
cd voxel-buddy
npm install
```

### Environment variables

Slack tokens can be supplied as environment variables as a fallback when no settings file exists yet. Values in the settings file always take precedence.

| Variable | Description |
|---|---|
| `SLACK_APP_TOKEN` | Slack App Token (`xapp-1-...`) |
| `SLACK_BOT_TOKEN` | Slack Bot Token (`xoxb-...`) |

All other credentials (Gmail OAuth, Claude API key) must be entered through the Settings UI.

### Development

```bash
# Recommended: start Vite dev server + Electron together
npm run dev:all

# Start only the Vite dev server (renderer hot-reload, no Electron)
npm run dev

# Start only Electron (assumes dev server already running on :5173)
npm run dev:electron
```

`npm run dev:all` uses `concurrently` and `wait-on` to launch the Vite dev server first, then start Electron only after `http://localhost:5173` is reachable. The renderer hot-reloads on source changes; the main process requires a manual restart.

### Running tests

```bash
npm test            # run once
npm run test:watch  # watch mode
```

## Slack App Setup

1. Go to [api.slack.com/apps](https://api.slack.com/apps) and click **Create New App → From scratch**.
2. Give it a name (e.g., "Voxel Buddy") and select your workspace.
3. Under **Settings → Socket Mode**, toggle Socket Mode **on**. When prompted, generate an **App-Level Token** with the `connections:write` scope. Copy this token — it starts with `xapp-1-`.
4. Under **Features → OAuth & Permissions → Bot Token Scopes**, add the following scopes:

   | Scope | Purpose |
   |---|---|
   | `connections:write` | App Token — required for Socket Mode |
   | `im:history` | Read direct messages |
   | `im:read` | List DM channels |
   | `users:read` | Resolve sender display names |
   | `chat:write` | Post replies back to Slack |

5. Click **Install to Workspace** and authorize. Copy the **Bot User OAuth Token** — it starts with `xoxb-`.
6. Under **Features → Event Subscriptions**, enable events and subscribe to these bot events:
   - `message.im` — direct messages to the bot
   - `app_mention` — @mentions of the bot in any channel
7. Open Voxel Buddy, right-click the mascot → **Settings**, paste both tokens into the Slack section, and click **Save**. Airie reconnects automatically.

## Gmail Setup

1. Open [Google Cloud Console](https://console.cloud.google.com) and create or select a project.
2. Go to **APIs & Services → Library**, search for **Gmail API**, and enable it.
3. Go to **APIs & Services → OAuth consent screen**. Choose **External** (or Internal for Google Workspace). Fill in the required fields. Add the scope `https://www.googleapis.com/auth/gmail.modify`.
4. Go to **APIs & Services → Credentials → Create Credentials → OAuth client ID**. Select **Desktop app** as the application type.
5. Copy the **Client ID** and **Client Secret**.
6. In Voxel Buddy Settings, under **Gmail**, click **Connect Gmail**, enter the Client ID and Client Secret, and click **Connect Gmail**. Your browser opens for Google consent; after approval the app captures the token automatically via the local callback server.

> The redirect URI used internally is `http://localhost:42813/oauth/google/callback`. For Desktop app credentials, Google permits localhost redirects without explicit registration.

## Building Distributables

```bash
npm run dist          # build for current platform

npm run dist:mac      # macOS DMG (arm64 + x64 universal)
npm run dist:win      # Windows NSIS installer (x64)
npm run dist:linux    # Linux AppImage
```

Built artifacts are placed in `release/`. The macOS build uses a hardened runtime with a custom entitlements file (`assets/entitlements.mac.plist`).
